# Metrics — Event Log & Snapshots

> 🌐 Leia esta documentação em [Português](README.md).

This directory stores local telemetry data for the Spec-Driven Agent.

## Format

### Raw Events (Source of Truth)

`events-YYYY-MM-DD.jsonl` — append-only, one file per day.

Each line is a JSON object with an automatic timestamp. It is securely built by `_utils.sh` → `event_logger()` → `json_build()` (uses `jq --arg`, no string interpolation — commands with quotes like `grep "pattern"` or `git commit -m "msg"` are properly escaped).

**Event types:**

| Event | Generated by | Core fields |
|--------|-----------|-------------------|
| `session_start` | `init-session.sh` | `session_id`, `project`, `model`, `mode` (LITE/FULL) |
| `session_end` | `stop.sh` | `session_id`, `reason` |
| `turn` | `pre-tool.sh` | `turn` (number), `phase`, `effort` |
| `tool` | `post-tool.sh` | `tool` (Read, Write, Edit, Bash...), `file` or `command` (truncated to 80 chars), `dur_ms`, `effort` |
| `agent` | `post-tool.sh` | `agent_type`, `model`, `dur_ms`, `effort`, `tokens` (object with `total`, `input`, `output`, `cache_write`, `cache_read`) |
| `task` | `post-task.sh` | `skill`, `spec`, `success` (boolean), `dur_s` |
| `gate` | `post-task.sh` / `check-gate.sh` | `gate` (spec/design/plan), `status`, `spec` |

### Snapshots (Optimized Read Cache)

Generated by `sda dashboard --build`. Files stored in `snapshots/`:

| File | Content |
|---------|----------|
| `daily-YYYY-MM-DD.json` | Daily aggregation (tasks, tokens, skills, agents, gates, economy) |
| `weekly-YYYY-WNN.json`  | Weekly aggregation |
| `total.json`            | Total aggregation |

Snapshots act as a read cache. The source of truth is always the JSONL files.
If a JSONL is newer than the snapshot, the dashboard detects it as stale (via `isSnapshotFresh()`) and automatically rebuilds it.

### Economy (LITE vs FULL)

The snapshot includes `economy` which compares token cost between modes:

| Field | Description |
|-------|-----------|
| `lite_tokens` | Total tokens used in LITE tasks |
| `full_baseline_per_agent` | Average tokens per agent call in FULL mode (baseline) |
| `estimated_full_tokens` | How much the LITE tasks would have cost in FULL mode |
| `saved_tokens` | Net token savings |
| `savings_pct` | Percentage of token savings |

Tracking is chronological: `tagEventsWithMode()` processes events in order, assigning each `agent`/`task` to the active mode at that moment (LITE or FULL).

## Lifecycle

- Events older than 90 days are compressed into a monthly snapshot.
- Compaction is automatic (triggered by the post-task hook, once per day).
- After compaction, the raw monthly JSONL file is removed.

## Privacy

**Zero telemetry.** All data remains in this local directory.
Nothing is sent to any external server.

## Commands

```bash
sda dashboard          # alias: sda metrics (same as --summary)
sda dashboard live     # Real-time TUI (readline, 1s polling)
sda dashboard summary  # Aggregated text summary
sda dashboard json     # Pure JSON output (pipe-friendly)
sda dashboard build    # Rebuilds snapshots from scratch
```
